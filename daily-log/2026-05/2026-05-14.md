## Stream API
- Java 8부터 도입된, 컬렉션·배열 등 데이터 소스를 함수형 스타일로 처리하기 위한 추상화 계층
- 데이터를 "어떻게(How)" 순회할지가 아니라 "무엇을(What)"할지 선언적으로 기술
- 내부 반복(Internal Iteration)을 사용하므로 JVM이 순회 전략·병렬화·최적화를 결정할 수 있음
- 핵심 특징
    - 선언적(Declarative): 의도를 코드에 그대로 드러냄
    - 합성 가능(Composable): 여러 연산을 파이프라인으로 연결
    - 지연 평가(Lazy): 최종 연산이 호출되기 전까지 중간 연산은 실행되지 않음
    - 불변(Immutable): 원본 데이터를 변경하지 않음
    - 일회성(Single-use): 한 번 소비된 Stream은 재사용 불가

### 명령형 vs. 선언형 비교
```java
// 명령형 (Imperative) -- "어떻게"
List<String> result = new ArrayList<>();
for (User user : users){
    if (user.getAge() >= 20){
        result.add(user.getName().toUpperCase());
    }
}
Collections.sort(result);

// 선언형 (Declarative) -- "무엇을"
List<String> result = users.stream()
    .filter(u -> u.getAge() >= 20)
    .map(u -> u.getName().toUpperCase())
    .sorted()
    .toList();
```

## Stream의 구조
> Stream은 소스(Source) → 중간 연산(Intermediate) → 최종 연산(Terminal)의 3단계 파이프라인으로 구성됨
```
[ Source ]      [ Intermediate Ops ]         [ Terminal Op ]
   ↓                  ↓    ↓    ↓                   ↓
Collection   →   filter → map → sorted   →   collect / forEach
 Array            (Lazy)  (Lazy) (Lazy)        (Eager, 실행 트리거)
 I/O Channel
 Generator
```

### Source(소스)
- Stream을 생성하는 시작점
- `Collection.stream()`, `Arrays.stream()`, `Stream.of()`, `Files.lines()`, `IntStream.range()` 등

### Intermediate Operation(중간 연산)
- Stream을 입력받아 새로운 Stream을 반환
- Lazy하므로 최종 연산이 없으면 실행되지 않음
- 예시: `filter`, `map`, `flatMap`, `sorted`, `distinct`, `limit`, `skip`, `peek`

### Terminal Operation(최종 연산)
- 파이프라인을 실행하고 결과를 반환
- 호출 시점에 모든 중간 연산이 한 번에 평가됨
- 예시: `collect`, `forEach`, `reduce`, `count`, `toList`, `findFirst`, `anyMatch`

## Stream 생성 방법
```java
// 1. Collection으로부터 생성
List<String> list = List.of("a", "b", "c");
Stream<String> s1 = list.stream();

// 2. 배열로부터 생성
String[] arr = {"a", "b", "c"};
Stream<String> s2 = Arrays.stream(arr);

// 3. 정적 메서드로 생성
Stream<String> s3 = Stream.of("a", "b", "c");

// 4. 무한 Stream (반드시 limit 필요)
Stream<Integer> s4 = Stream.iterate(0, n -> n + 2).limit(10);
Stream<Double> s5 = Stream.generate(Math::random).limit(5);

// 5. 기본형(Primitive) Stream
IntStream s6 = IntStream.rangeClosed(1, 100);
IntStream s7 = IntStream.range(1, 100);

// 6. 파일 라인을 Stream으로
try (Stream<String> lines = Files.lines(Paths.get("data.txt"))) {
    lines.filter(l -> !l.isBlank()).forEach(System.out::println);
}
```
> `Files.lines`처럼 시스템 자원을 사용하는 Stream은 반드시 try-with-resources로 감싸 닫아야 함

## 주요 중간 연산
### `filter` -- 조건 필터링
```java
List<User> adults = users.stream()
    .filter(u -> u.getAge() >= 20)
    .toList();
```

### `map`/`mapToInt` -- 요소 변환
```java
List<String> names = users.stream()
    .map(User::getName)
    .toList();

// 박싱 비용을 피하려면 기본형 Stream으로 변환
int totalAge = users.stream()
    .mapToInt(User::getAge)     // Stream<User> -> IntStream
    .sum();
```

### `flatMap` -- 중첩 구조 평탄화
```java
List<List<Integer>> nested = List.of(
    List.of(1, 2, 3),
    List.of(4, 5),
    List.of(6, 7, 8)
);

List<Integer> flat = nested.stream()
    .flatMap(List::stream)
    .toList();  // [1, 2, 3, 4, 5, 6, 7, 8]
```
- `map`은 `Stream<Stream<T>>`를 만들지만, `flatMap`은 이를 단일 `Stream<T>`로 펼침
- 1:N 관계의 엔티티(예: User → Orders)를 한 번에 처리할 때 자주 사용

### `sorted`/`distinct`, `limit`, `skip`
```java
List<Integer> top3 = numbers.stream()
    .distinct()                             // 중복 제거
    .sorted(Comparator.reverseOrder())      // 내림차순 정렬
    .limit(3)                               // 상위 3개
    .toList();
```

### `peek` -- 디버깅용
```java
List<String> result = users.stream()
    .peek(u -> log.debug("before filter: {}", u))
    .filter(u -> u.getAge() >= 20)
    .peek(u -> log.debug("after filter: {}", u))
    .map(User::getName)
    .toList();
```
> ⚠️ `peek`은 디버깅 목적임. 부수효과를 위한 용도로 쓰면 안됨

## 주요 최종 연산
### `collect` -- 가장 강력한 수집 연산
```java
// List, Set, Map으로 수집
List<String> list = stream.collect(Collectors.toList());
Set<String> set = stream.collect(Collectors.toSet());
Map<Long, User> map = users.stream()
    .collect(Collectors.toMap(User::getId, Function.identity()));

// 그룹화 (Group By)
Map<String, List<User>> byCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity));

// 그룹화 + 집계
Map<String, Long> countByCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity, Collectors.counting()));

Map<String, Double> avgAgeByCity = users.stream()
    .collect(Collectors.groupingBy(
        User::getCity,
        Collectors.averagingInt(User::getAge)
    ));

// 분할 (Partition By) -- key가 boolean
Map<Boolean, List<User>> partitioned = users.stream()
    .collect(Collectors.partitioningBy(u -> u.getAge() >= 20));

// 문자열 결합
String joined = users.stream()
    .map(User::getName)
    .collect(Collectors.joining(", ", "[", "]"));
```

### `reduce` -- 누적 연산
```java
// 단일 항등원 사용
int sum = numbers.stream().reduce(0, Integer::sum);

// 항등원 없이 (Optional 반환)
Optional<Integer> max = numbers.stream().reduce(Integer::max);

// 3-인자 reduce (병렬 처리용)
int totalAge = users.parallelStream().reduce(
    0,                              // identity
    (acc, u) -> acc + u.getAge(),   // accumulator
    Integer::sum                    // combiner (병렬 시 부분 합 결합)
);
```

### `findFirst`/`findAny`/`anyMatch`/`allMatch`/`noneMatch`
```java
Optional<User> first = users.stream()
    .filter(u -> u.getAge() >= 20)
    .findFirst();

boolean hasAdult = users.stream().anyMatch(u -> u.getAge() >= 20);
boolean allAdult = users.stream().allMatch(u -> u.getAge() >= 20);
```
> `findAny`는 병렬 Stream에서 더 효율적임. 순서가 보장되지 않아도 되는 경우 권장

### `toList()`(Java 16+)
```java
// 권장 -- 불변 List 반환
List<String> names = users.stream().map(User::getName).toList();

// 기존 방식 -- 가변 List 반환
List<String> names = users.stream().map(User::getName).collect(Collectors.toList());
```

## Lazy Evaluation(지연 평가)
> Stream의 핵심 동작 원리. 중간 연산은 최종 연산이 호출되는 순간에 비로소 실행됨
```java
Stream<Integer> stream = Stream.of(1, 2, 3, 4, 5)
    .filter(n -> {
        System.out.println("filter: " + n);
        return n % 2 == 0;
    })
    .map(n -> {
        System.out.println("map: " + n);
        return n * n;
    });

System.out.println("--- 최종 연산 호출 ---");
List<Integer> result = stream.toList();
```
- 출력
    ```
    --- 최종 연산 호출 ---
    filter: 1
    filter: 2
    map: 2
    filter: 3
    filter: 4
    map: 4
    filter: 5
    ```
- 요소 하나하나가 수직(vertical)으로 파이프라인을 통과함
    - 단순한 for-each처럼 `filter` 전체 → `map` 전체 순으로 처리되지 않음
- 덕분에 `limit`, `findFirst` 같은 단락(short-circuit) 연산에서 불필요한 처리를 건너뜀

### Short-circuit 예시
```java
Optional<Integer> first = Stream.iterate(1, n -> n + 1)     // 무한 Stream
    .filter(n -> n > 1000)
    .findFirst();       // 1001을 찾는 즉시 종료
```

## 기본형(Primitive) Stream
- `Stream<Integer>`처럼 박싱된 Stream은 오토박싱·언박싱 비용이 큼
- 숫자 처리에는 `IntStream`, `LongStream`, `DoubleStream`을 사용

|   구분    |   Stream<Integer> |   IntStream   |
|----------|--------------------|---------------|
|   요소 타입   |   `Integer`(객체) |   `int`(기본형)   |
|   박싱 비용   |   매번 발생   |   없음    |
|   집계 메서드 |   `reduce`로 직접 구현    |   `sum`, `average`, `max`, `min` 등 내장  |
|   변환    |   `mapToInt`로 IntStream으로 변환 |   `boxed`로 Stream<Integer>로 변환    |

```java
// 1부터 100까지의 합
int sum = IntStream.rangeClosed(1, 100).sum();

// User 리스트의 평균 나이
OptionalDouble avg = users.stream()
    .mapToInt(User::getAge)
    .average();

// 통계 한 번에 계산
IntSummaryStatistics stats = users.stream()
    .mapToInt(User::getAge)
    .summaryStatistics();

stats.getMin();
stats.getMax();
stats.getAverage();
stats.getSum();
stats.getCount();
```

## 병렬 Stream(`parallelStream`)
- 내부적으로 Fork/Join Framework와 `ForkJoinPool.commonPool()`을 사용해 작업을 분할 처리
- `parallel()` 또는 `parallelStream()`만 호출하면 즉시 병렬화됨
```java
long count = users.parallelStream()
    .filter(u -> u.getAge() >= 20)
    .count();
```

### 병렬 Stream을 사용해도 되는 조건
1. 데이터 크기가 충분히 큼(수만~수십만 이상)
2. 각 연산이 CPU 바운드이고 상태가 없음(stateless)
3. 분할이 효율적인 소스(`ArrayList`, 배열 ⭕/`LinkedList`, `Iterator` ❌)
4. 합성(combiner)이 결합 법칙(associative)을 만족함

### 병렬 Stream을 피해야 하는 경우
> ⚠️ 잘못 사용하면 싱글 스레드보다 더 느려지거나, 정확성 자체가 깨짐
- I/O 바운드 작업(DB 호출, 외부 API 호출 등)
- 공유 상태를 변경하는 코드(Race Condition 발생)
```java
// ❌ 절대 금지 -- shared mutable state
List<Integer> result = new ArrayList<>();
numbers.parallelStream().forEach(result::add);  // 동시성 문제 발생

// ✅ 권장 -- collect 사용
List<Integer> result = numbers.parallelStream().collect(Collectors.toList());
```

- 웹 서버 환경
    - `commonPool`은 JVM 전역에서 공유되므로, 한 요청의 병렬 Stream이 다른 요청을 지연시킬 수 있음
    - 별도 `ForkJoinPool`을 만들어 격리하거나, 가급적 병렬 Stream 대신 비동기 처리(`CompletableFuture`)를 권장

## 주의사항 및 안티 패턴
### ⚠️ Stream은 일회용
```java
Stream<String> stream = list.stream();
stream.forEach(System.out::println);
stream.count();     // ❌ IllegalStateException: stream has already been operated upon or closed
```

### ⚠️ Stream 내부에서 외부 변수 수정 금지
```java
// ❌ Bad -- 부수효과 발생
List<String> result = new ArrayList<>();
users.stream()
    .filter(u -> u.getAge() >= 20)
    .forEach(u -> result.add(u.getName()));

// ✅ Good -- collect 사용
List<String> result = users.stream()
    .filter(u -> u.getAge() >= 20)
    .map(User::getName)
    .toList();
```

### ⚠️ 단순 반복은 for-each가 더 빠르고 명확함
```java
// 단순 순회만 한다면 굳이 Stream을 쓸 필요 없음
for (User user : users){
    sendEmail(user);
}
```
- Stream은 변환·필터링·집계 등의 데이터 처리에 강점이 있음
- 단순 순회·외부 시스템 호출에는 일반 반복문이 더 적절함

### ⚠️ Checked Exception 처리 곤란
- Stream의 람다는 Checked Exception을 던질 수 없음
- 해결책
    - `try-catch`로 람다 내부에서 처리
    - `RuntimeException`으로 래핑
    - 별도 유틸리티 메서드로 추출

```java
List<String> contents = path.stream()
    .map(path -> {
        try {
            return Files.readString(path);
        } catch (IOException e){
            throw new UncheckedIOException(e);
        }
    })
    .toList();
```

## 성능 고려사항
|   상황    |   권장 방식   |
|----------|--------------|
|   소규모 데이터(수십~수백)    |   일반 반복문 또는 Stream |
|   숫자 집계   |   기본형 Stream(`IntStream` 등)   |
|   대용량 CPU 바운드 처리  |   `parallelStream` 검토   |
|   I/O 바운드 작업 |   `CompletableFuture` 등 비동기 처리  |
|   복잡한 비즈니스 로직    |   가독성 위해 Stream 분리·메서드 추출 |

> Stream은 함수 호출 오베헤드와 객체 생성 비용이 있어 매우 단순한 작업에서는 일반 반복문보다 약간 느릴 수 있음
>
> 다만 대부분의 백엔드 코드에서 이 차이는 무시할 만한 수준이며, 가독성과 유지보수성의 이점이 훨씬 큼
