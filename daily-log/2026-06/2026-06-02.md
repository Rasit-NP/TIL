## 컴파일러가 자동으로 합성하는 멤버 함수
- C++에서는 클래스를 정의할 때 개발자가 명시하지 않아도 컴파일러가 여섯 가지 멤버 함수를 자동으로 생성함
- C++11 이전에는 네 가지, C++11 이후로는 이동 생성자와 이동 대입 연산자가 추가됨

```cpp
class Resource{
public:
    Resource();                                     // 기본 생성자
    ~Resource();                                    // 소멸자

    Resource(const Resource& other);                // 복사 생성자
    Resource& operator=(const Resource& rhs);       // 복사 대입 연산자

    Resource(Resource&& other) noexcept;            // 이동 생성자
    Resource& operator=(Resource&& rhs) noexcept;   // 이동 대입 연산자
};
```

### 자동 생성 억제 규칙
- 특수 멤버 함수들은 서로의 선언 여부에 따라 자동 생성이 억제되거나 삭제됨
| 사용자가 직접 선언한 함수 | 이동 생성자 | 이동 대입 연산자 |
|-----------------------|------------|---------------|
| 없음 | 자동 생성 | 자동 생성 |
| 소멸자만 선언 | 생성 안 됨 | 생성 안 됨 |
| 복사 생성자만 선언 | 생성 안 됨 | 생성 안 됨 |
| 이동 생성자만 선언 | 사용자 정의 | 생성 안 됨 |

- 소멸자 하나만 선언해도 이동 연산이 자동 생성되지 않으므로, 이동을 원한다면 반드시 명시해야 함

## Rule of Five
- 동적 메모리, 파일 핸들 등 자원을 직접 소유하는 클래스는 소멸자·복사 생성자·복사 대입 연산자·이동 생성자·이동 대입 연산자, 다섯 가지를 모두 명시적으로 정의해야 함
- 다섯 중 하나라도 직접 작성했다면, 나머지 넷도 함께 작성해야 올바른 동작을 보장할 수 있음
```cpp
class Buffer {
    int*    data_;
    size_t  size_;
public:
    // 1. 소멸자
    ~Buffer() { delete[] data_; }

    // 2. 복사 생성자 — 깊은 복사
    Buffer(const Buffer& other)
        : data_(new int[other.size_]), size_(other.size_){
        std::copy(other.data_, other.data_+size_, data_);
    }

    // 3. 복사 대입 연산자 — 깊은 복사 + 자기 대입 방어
    Buffer& operator=(const Buffer& rhs){
        if (this == &rhs) return *this;
        delete[] data_;
        data_ = new int[rhs.size_];
        size_ = rhs.size_;
        std::copy(rhs.data_, rhs.data_+size_, data_);
        return *this;
    }

    // 4. 이동 생성자 — 소유권 이전
    Buffer(Buffer&& other) noexcept
        : data_(other.data_), size_(other.size_){
        other.data_ = nullptr;
        other.size_ = 0;
    }

    // 5. 이동 대입 연산자 — 소유권 이전
    Buffer& operator=(Buffer&& rhs) noexcept {
        if (this == &rhs) return *this;
        delete[] data_;
        data_ = rhs.data_;
        size_ = rhs.size_;
        rhs.data_ = nullptr;
        rhs.size_ = 0;
        return *this;
    }
};
```

### Rule of Zero
- 반대로, 자원을 직접 소유하지 않는 클래스는 다섯 가지 연산을 모두 선언하지 않는 것이 권장됨
- 자원 관리를 `std::unique_ptr`, `std::vector`처럼 RAII를 구현한 타입에 위임하면 컴파일러가 자동 생성한 함수들이 올바르게 동작함

```cpp
// Rule of Zero — 특수 멤버 함수 선언 없음
class Config{
    std::string             name_;
    std::vector<int>        values_;
    std::unique_ptr<Logger> logger_;
    // unique_ptr이 복사 금지·이동 허용을 자동으로 표현함
};
```

## 이동 시맨틱스
- C++11에서 도입된 이동 시맨틱스는 자원의 소유권을 복사 없이 이전하는 언어 수준의 메커니즘임
- 컨테이너 재할당, 함수 반환, 임시 객체 처리 등 곳곳에서 깊은 복사를 제거해 성능을 끌어올림

### lvalue·rvalue와 값 범주
- C++의 모든 표현식은 값 범주(value category)를 가짐
- lvalue는 이름을 가지며 주소를 얻을 수 있는 표현식임
- rvalue는 이름이 없는 임시 객체이거나 주소를 얻을 수 없는 표현식임

```cpp
int x = 10;
// x는 lvalue(이름 있음), 10은 rvalue(리터럴, 이름 없음)

Buffer b1;
Buffer b2 = makeBuffer();   // makeBuffer()의 반환값은 rvalue
```

### `std::move`와 소유권 이전
- `std::move`는 이동을 직접 수행하는 함수가 아니라, lvalue를 rvalue 참조로 캐스팅하는 캐스트임
- 이동 생성자 또는 이동 대입 연산자의 호출을 유도하는 역할만 함
- `std::move` 이후 원본 객체는 "유효하지만 내용을 불확정"(valid but unspecified) 상태가 됨
```cpp
std::vector<int> a = {1, 2, 3, 4, 5};

// 복사  — Heap에 새 배열 할당 후 원소 하나씩 복사, O(n)
std::vector<int> b = a;

// 이동 — 내부 포인터만 빼앗아 옴, O(1)
std::vector<int> c = std::move(a);
// 이후 a.size() == 0; a의 내부 포인터는 nullptr로 교체됨
```

### `noexcept`의 역할
- 이동 생성자와 이동 대입 연산자에 `noexcept`를 명시하지 않으면 `std::vector`가 재할당 시 복사를 선택함
- `std::vector`는 예외 안정성을 보장하기 위해, 이동이 예외를 던질 수 있다고 판단되면 안전한 복사로 폴백(fallback)하기 때문
- 이동 연산에 `noexcept`가 없으면 `vector`에 보관된 타입의 재할당이 $O(1)$이 아닌 $O(n)$으로 처리될 수 있음
```cpp
class Buffer{
public:
    Buffer(Buffer&& other) noexcept;            // noexcept 없으면 vector가 복사 선택
    Buffer& operator=(Buffer&& rhs) noexcept;
};

std::vector<Buffer> v;
v.push_back(Buffer{});  // noexcept 명시 시 이동, 미명시 시 복사
```
