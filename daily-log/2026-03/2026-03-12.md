## Prometheus
- 대상 시스템으로부터 지표(Metrics)를 수집하여 저장하고, 이에 대한 질의 및 알람 기능을 제공하는 오픈소스 모니터링 시스템

### 핵심 동작 원리 및 아키텍처
- Pull 방식의 데이터 수집
    - Prometheus 서버가 설정된 주기(Scrape Interval)마다 대상 시스템의 HTTP 엔드포인트에 직접 접속하여 데이터를 긁어옴
    - 대상 시스템이 데이터를 밀어넣는 Push 방식에 비해 서버의 부하 제어가 용이하고 대상의 생존 여부를 즉각 확인하기 유리함
- 시계열 데이터베이스(TSDB)
    - 모든 메트릭은 시간의 흐름에 따른 상태 변화(Time-series)로 저장됨
    - 데이터는 `Metric Name`과 `Labels`(Key-Value 쌍)의 조합으로 식별되어 다차원적인 질의가 가능함
- 서비스 디스커버리(Service Discovery)
    - 정적인 타겟 설정뿐만 아니라, Kubernetes나 가상화 환경에서 동적으로 변하는 컨테이너들의 IP 주소를 자동으로 탐색하고 수집 대상으로 등록하는 유연한 기능을 제공함

### 데이터 모델(Metric Types)
1. Counter
    - 누적되는 값으로, 오직 증가하거나 0으로 초기화만 가능함
    - 예: 시스템 시작 이후 총 HTTP 요청 횟수, 발생한 에러의 총 합계
2. Gauge
    - 현재 시점의 상태를 나타내며, 자유롭게 증가하거나 감소할 수 있는 지표
    - 예: 현재 사용 중인 메모리의 양, 활성 스레드 수, 동시 접속자 수
3. Histogram
    - 데이터의 분포 범위를 관찰함
    - 미리 설정된 버킷(Bucket) 별 빈도를 기록하여 응답시간의 95th Percentile 등을 계산할 때 필수적
4. Summary
    - Histogram과 유사하지만, 클라이언트 측에서 미리 분위수(Quantile)를 계산하여 기록함

## Grafana
- 다양한 데이터 소스로부터 수집된 데이터를 시각화, 분석 및 모니터링하기 위한 전문 대시보드

### 핵심 구성 요소
- Data Source
    - 데이터를 공급하는 백엔드 저장소
    - 본 시스템에서는 Prometheus가 메인 소스로 활용됨
- Dashboard
    - 여러 시각화 패널의 집합체로, 전체적인 시스템 상태를 한 눈에 파악할 수 있는 캔버스
    - JSON 포맷으로 추출 및 임포트가 가능
- Panel
    - 개별 그래프, 표, 히트맵, 게이지 등을 나타내는 시각화의 최소 단위
    - 각 패널은 독립적인 질의(Query)를 가짐
- Variables
    - 대시보드 상단에서 특정 서버명이나 데이터베이스 인스턴스를 선택하여 데이터를 실시간으로 필터링할 수 있게 하는 기능

### 시각화 프로세스
1. Query
    - 사용자가 Grafana 패널 편집 창에서 Prometheus 전용 질의어인 PromQL을 작성
2. Request
    - Grafana는 설정된 데이터 소스 설정값을 바탕으로 Prometheus 서버의 API 엔드포인트에 질의를 전송
3. Visualization
    - Prometheus로부터 수신된 시계열 데이터 결과값을 Grafana 엔진이 해석하여 사용자가 정의한 차트, 히트맵 등으로 브라우저에 시각화

## 구성 요소별 상세 설정
### Spring Boot
- Spring Boot의 내부 지표(JVM, 힙 메모리, HTTP 요청 등)를 Prometheus 포맷으로 노출하기 위한 과정
- `build.gradle` 의존성 추가
    ```gradle
    dependencies {
    // 런타임 중 애플리케이션 상태 확인 및 메트릭 수집을 위한 Actuator
    implementation 'org.springframework.boot:spring-boot-starter-actuator'
    // Actuator 데이터를 Prometheus가 해석 가능한 텍스트 포맷으로 변환해주는 레지스트리
    runtimeOnly 'io.micrometer:micrometer-registry-prometheus'
    }
    ```
- `application.yml` 설정
    ```yaml
    management:
        endpoints:
            web:
            exposure:
                include: prometheus # /actuator/prometheus 엔드포인트 활성화
        endpoint:
            prometheus:
                enabled: true
        metrics:
            tags:
                application: ${spring.application.name} # 대시보드 필터링을 위한 애플리케이션 태그 추가
    ```

### PostgreSQL 모니터링 설정(Exporter)
- PostgreSQL은 자체적으로 Prometheus 포맷을 지원하지 않기 때문에, 데이터베이스와 Prometheus 사이를 연결하는 가교 역할의 Exporter가 필수임
- 필요성
    - PostgreSQL 내부 통계 테이블(`pg_stat_activity`, `pg_stat_database` 등)의 데이터를 질의
    - Prometheus가 이해할 수 있는 시계열 메트릭 포맷으로 변환해 노출
- 수집 지표
    - 활성 연결 수, 쿼리 처리량, 데이터베이스 크기, 캐시 히트율, 락(Lock) 대기 상태 등

### Nginx 모니터링 설정
- Nginx 웹 서버의 트래픽 상태를 모니터링하기 위해 `stub_status` 모듈을 활성화하고 전용 Exporter를 사용
- `nginx.conf` 설정 수정
    ```nginx
    server{
        listen 443;
        server_name <domain>;
        
        # Nginx Exporter가 접근할 전용 엔드포인트
        location /nginx_status {
            stub_status on;
            access_log off;     # 불필요한 로그 생성 방지
            allow 127.0.0.1;    # 로컬 접근 허용
            allow 172.16.0.0/12; # Docker 컨테이너 간 통신 허용 (네트워크 대역에 맞춰 조정)
            deny all;           # 그 외 모든 접근 차단
        }
    }
    ```
    - Nginx 설정 파일 내부의 `server` 블록에 위 내용을 추가하여 상태 노출 엔드포인트를 생성

## 인프라 정의 및 시스템 연동
### `prometheus.yml`
```yaml
global:
  scrape_interval: 15s     # 전체 메트릭 수집 주기
  evaluation_interval: 15s # 알람 규칙 평가 주기

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'spring-boot-app'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['app-container:8080']

  - job_name: 'postgres'
    static_configs:
      - targets: ['postgres-exporter:9187']

  - job_name: 'nginx'
    static_configs:
      - targets: ['nginx-exporter:9113']
```
- 모든 수집 대상을 통합 정의

### `docker-compose.monitor.yml` 작성
```yaml
version: '3.8'

services:
  prometheus:
    image: prom/prometheus:latest
    container_name: monitoring-prometheus
    restart: unless-stopped
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - prometheus-data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=15d' # 15일 보관 정책
    ports:
      - "9090:9090"
    networks:
      - app-network

  grafana:
    image: grafana/grafana:latest
    container_name: monitoring-grafana
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - grafana-data:/var/lib/grafana
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=admin
    networks:
      - app-network
    depends_on:
      - prometheus

  postgres-exporter:
    image: prometheuscommunity/postgres-exporter:latest
    container_name: postgres-exporter
    restart: unless-stopped
    environment:
      # 포맷: postgresql://사용자:비밀번호@호스트:포트/DB명?sslmode=disable
      - DATA_SOURCE_NAME=postgresql://postgres:password@postgres-db:5432/postgres?sslmode=disable
    networks:
      - app-network

  nginx-exporter:
    image: nginx/nginx-prometheus-exporter:latest
    container_name: nginx-exporter
    restart: unless-stopped
    command:
      # Nginx 컨테이너의 stub_status 주소 명시
      - '-nginx.scrape-uri=http://nginx-container:80/stub_status'
    networks:
      - app-network

networks:
  app-network:
    external: true # 기존 프로젝트의 네트워크명을 여기에 작성

volumes:
  prometheus-data:
  grafana-data:
```
- 기존 프로젝트의 네트워크를 외부(external)로 설정하여 통합

## 검증 및 운영

```bash
docker compose -f docker-compose.monitor.yml up -d
```

### 모니터링 대상 상태 확인
- `http://<domain>:9090/targets`에 접속하여 다음 4개 항목이 모두 **UP** 상태인지 확인
    - prometheus
    - spring-boot-app
    - postgres
    - nginx
