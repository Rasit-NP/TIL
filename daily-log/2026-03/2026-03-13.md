# 2026-03-13

## 🧠 오늘 배운 것
- Trouble Shooting
    - Spring Boot 4.0 모니터링 수집 불가 장애

### 1. 장애 현상
- Grafana
    - Spring Boot 관련 대시보드에서만 데이터가 표기되지 않음
- 검증
    ```bash
    # Prometheus 컨테이너 접속 후 수행
    $ docker exec -it <prometheus_container_id> sh
    $ wget -qO- http://backend:8080/actuator/prometheus
    ```
    - Prometheus 컨테이너 내부에서 Spring Boot 엔드포인트 호출 시 보안 거부 발생

### 2. 환경 정보
- Spring Boot 4.0
- Spring Boot Starter Actuator, Micrometer Registry Prometheus
- Docker Compose

### 3. 원인 분석
- Spring Boot 4.0은 애플리케이션의 모든 엔드포인트에 대해 기본적으로 인증을 요구함
- 타 서비스(Redis, Nginx 등)는 별도의 익스포터를 거치므로 수집에 문제가 없으나, Spring Boot는 내부 Actuator가 직접 통신을 담당함
- 이 때, Spring Security가 Prometheus의 익명 접근을 차단하여 HTTP 403 Forbidden을 반환

### 4. 해결 방법
- Spring Security 설정 수정
    ```java
    @Configuration
    @EnableWebSecurity
    public class SecurityConfig {
        @Bean
        public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
            http
                .csrf(csrf -> csrf.disable())
                .authorizeHttpRequests(auth -> auth
                    .requestMatchers("/actuator/prometheus", "/actuator/health").permitAll()
                    .anyRequest().authenticated()
                );
            return http.build();
        }
    }
    ```
    - Prometheus 수집 경로(`/actuator/prometheus`)에 대한 보안 필터를 해제
- Actuator 노출 설정 확인(`application.yml`)
    ```yaml
    management:
      endpoints:
        web:
          exposure:
            include: prometheus, health, info
      endpoint:
        prometheus:
          enabled: true
      metrics:
        tags:
          application: ${spring.application.name}
    ```
    - 보안 해제와 더불어, 엔드포인트가 웹으로 노출되도록 명시되어야 함

### 검증 결과
1. 애플리케이션 재빌드 및 배포
    - 수정된 `SecurityConfig`를 적용하여 컨테이너를 재시작
2. 접근 테스트
    ```bash
    /prometheus $ wget -qO- http://backend:8080/actuator/prometheus
    ```
    - Prometheus 컨테이너 내부에서 다시 호출을 수행
3. Grafana 확인
    - 대시보드에 그래프가 정상적으로 그려지는지 확인
