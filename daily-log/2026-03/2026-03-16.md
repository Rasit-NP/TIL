## C++ 상호 참조 구조체
### 문제 상황
```C++
struct A {
    B* next;
};

struct B {
    A* prev;
};
```
- 두 구조체가 서로를 참조할 때, 단순히 선언 순서대로 쓰면 컴파일 에러가 발생함
    - `B`가 완전한 타입(complete type)이 아닌 상태에서 멤버로 사용되면 컴파일러가 크기를 알 수 없어 실패

### 전방 선언(Forward Declaration)
```C++
struct B;       // 전방 선언 - "B"라는 구조체가 있다"는 사실만 알림

struct A {
    B* next;    // OK: 포인터는 온전한 타입 불필요
    B& ref;     // OK: 참조도 마찬가지
    B val;      // 여전히 오류 발생(변수 할당을 위한 크기를 알아야 함)
};

struct B {
    A* prev;
};
```
- 포인터나 참조로 사용할 때는 **완전한 정의 없이 이름만 미리 선언**해두면 충분
- 컴파일러는 포인터 크기만 알면 되기 때문
- 메서드 구현에서 전방 선언된 타입의 멤버에 접근하려면, 해당 타입이 완전히 정의된 **이후**에 구현해야 함

### `inline` 구현
```C++
struct B;

struct A {
    void greet(const B& b);   // 선언만
};

struct B {
    string name;
    void link(const A& a);    // 선언만
};

inline void A::greet(const B& b){
    b.name;
}

inline void B::link(const A& a){
    // a의 멤버 접근 가능
}
```

## DockerHub + Jenkins
### 사전 준비
1. 필요한 계정 및 환경
    - **Docker Hub** 계정
        - hub.docker.com에서 무료 가입
    - **GitHub/GitLab**
        - 소스코드 저장소
    - 서버 2대
        - `Jenkins 서버`
            - Jenkins 설치용
        - `배포 서버`
            - 실제 애플리케이션이 실행될 서버
2. Jenkins 서버에 Docker 설치
    ```bash
    # Ubuntu 기준
    sudo apt-get update
    sudo apt-get install -y ca-certificates curl gnupg lsb-release

    # Docker 공식 GPG 키 추가
    sudo mkdir -p /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
        | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

    # Docker repository 등록
    echo \
    "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" \
    | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    sudo apt-get update
    sudo apt-get install -y docker-ce docker-ce-cli containerd.io

    # jenkins 유저가 docker 명령어를 쓸 수 있도록 그룹 추가
    sudo usermod -aG docker jenkins
    sudo systemctl restart jenkins
    ```

### Jenkins 설치 및 초기 설정
1. Jenkins 설치
    ```bash
    # Java 설치 (Jenkins 실행 필수)
    sudo apt-get install -y openjdk-17-jdk

    # Jenkins 저장소 추가
    curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key \
    | sudo tee /usr/share/keyrings/jenkins-keyring.asc > /dev/null

    echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
    https://pkg.jenkins.io/debian-stable binary/ \
    | sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null

    sudo apt-get update
    sudo apt-get install -y jenkins

    # 서비스 시작
    sudo systemctl enable jenkins
    sudo systemctl start jenkins

    # 초기 비밀번호 확인
    sudo cat /var/lib/jenkins/secrets/initialAdminPassword
    ```
2. Jenkins 웹 UI 접속
    1. `http://<서버IP>:8080`으로 접속
    2. 초기 비밀번호 입력
    3. **Install suggested plugins** 선택
    4. 관리자 계정 생성
3. 필수 플러그인 설치
    - Jenkins 관리 > Plugins > Available plugins에서 아래를 설치
    | 플러그인 | 용도 |
    |---|---|
    | `Docker Pipeline` | Docker 빌드/Push 파이프라인 지원 |
    | `Docker Commons` | Docker 관련 공통 기능 |
    | `Git` | GitHub 연동 |
    | `GitHub Integration` | Webhook 자동 트리거 |
    | `SSH Agent` | 원격 서버 SSH 접속 |
    | `Credentials Binding` | 비밀번호/토큰 안전 관리 |
    | `Pipeline` | Jenkinsfile 파이프라인 실행 |

### Credentials 등록
- Jenkins에서 Docker Hub 비밀번호, SSH 키 등 민감 정보를 안전하게 관리
- Jenkins 관리 > Security > Credentials > System > Global credentials
1. Docker Hub 계정 등록
    ```
    Kind     : Username with password
    Username : <Docker Hub 아이디>
    Password : <Docker Hub 패스워드 또는 Access Token>
    ID       : dockerhub-credentials   ← Jenkinsfile에서 이 ID로 참조
    ```
    - Docker Hub Access Token 발급 권장
        - Docker Hub > Account Settings > Security > New Access Token
2. 배포 서버 SSH 키 등록
    ```bash
    # Jenkins 서버에서 SSH 키페어 생성
    ssh-keygen -t rsa -b 4096 -C "jenkins@cicd" -f ~/.ssh/deploy_key

    # 공개키를 배포 서버에 등록
    ssh-copy-id -i ~/.ssh/deploy_key.pub user@<배포서버IP>
    ```
    - Jenkins Credentials에 아래 정보를 추가
        ```
        Kind        : SSH Username with private key
        ID          : deploy-server-ssh
        Username    : ubuntu  (배포 서버 계정명)
        Private Key : Enter directly → 위에서 생성한 개인키 내용 붙여넣기
        ```
3. GitHub Token 등록(Webhook 용)
```
Kind   : Secret text
Secret : <GitHub Personal Access Token>
ID     : github-token
```

### Dockerfile 작성
- 프로젝트 루트에 `Dockerfile`을 생성
```dockerfile
# ── Build stage ──────────────────────────────────────
FROM eclipse-temurin:17-jdk-alpine AS builder
WORKDIR /app

COPY gradlew .
COPY gradle gradle
COPY build.gradle settings.gradle .
RUN ./gradlew dependencies --no-daemon

COPY src src
RUN ./gradlew bootJar --no-daemon

# ── Runtime stage ─────────────────────────────────────
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app

RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

COPY --from=builder /app/build/libs/*.jar app.jar

EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Jenkinsfile 작성
- 프로젝트 루트에 `Jenkinsfile`을 생성
```groovy
pipeline {
    agent any

    // ── 환경변수 ────────────────────────────────────
    environment {
        DOCKER_HUB_REPO = 'your-dockerhub-username/your-app-name'
        DEPLOY_SERVER   = 'ubuntu@<배포서버IP>'
        CONTAINER_NAME  = 'my-app'
        APP_PORT        = '8080'
        IMAGE_TAG       = "${env.BUILD_NUMBER}"
    }

    // ── 빌드 트리거 ─────────────────────────────────
    triggers {
        githubPush()
    }

    stages {

        stage('Checkout') {
            steps {
                checkout scm
                echo "Branch: ${env.GIT_BRANCH}"
                echo "Commit: ${env.GIT_COMMIT}"
            }
        }

        stage('Test') {
            steps {
                sh './gradlew test --no-daemon'
            }
            post {
                always {
                    junit 'build/test-results/**/*.xml'
                }
            }
        }

        stage('Docker Build') {
            steps {
                script {
                    dockerImage = docker.build("${DOCKER_HUB_REPO}:${IMAGE_TAG}")
                    docker.build("${DOCKER_HUB_REPO}:latest")
                }
            }
        }

        stage('Docker Push') {
            steps {
                script {
                    docker.withRegistry('https://registry.hub.docker.com',
                                        'dockerhub-credentials') {
                        dockerImage.push("${IMAGE_TAG}")
                        dockerImage.push("latest")
                    }
                }
            }
        }

        stage('Deploy') {
            when {
                branch 'main'
            }
            steps {
                sshagent(['deploy-server-ssh']) {
                    sh """
                        ssh -o StrictHostKeyChecking=no ${DEPLOY_SERVER} '
                            docker pull ${DOCKER_HUB_REPO}:latest
                            docker stop ${CONTAINER_NAME} || true
                            docker rm   ${CONTAINER_NAME} || true
                            docker run -d \
                                --name ${CONTAINER_NAME} \
                                --restart unless-stopped \
                                -p ${APP_PORT}:${APP_PORT} \
                                ${DOCKER_HUB_REPO}:latest
                            docker image prune -f
                        '
                    """
                }
            }
        }

        stage('Health Check') {
            when {
                branch 'main'
            }
            steps {
                script {
                    sleep(time: 15, unit: 'SECONDS')
                    def response = sh(
                        script: "curl -s -o /dev/null -w '%{http_code}' http://<배포서버IP>:${APP_PORT}/health",
                        returnStdout: true
                    ).trim()
                    if (response != '200') {
                        error "헬스체크 실패 (HTTP ${response})"
                    }
                    echo "헬스체크 성공: HTTP ${response}"
                }
            }
        }
    }

    post {
        success {
            echo "✅ 배포 성공: ${DOCKER_HUB_REPO}:${IMAGE_TAG}"
        }
        failure {
            echo "❌ 파이프라인 실패"
        }
        always {
            cleanWs()
        }
    }
}
```

### Jenkins Pipeline Job 생성
1. Jenkins 메인 > **New Item**
2. 이름 입력 > **Pipeline** 선택 > OK
3. 설정 화면에서 아래와 같이 입력
    ```
    General
    ☑ GitHub project
    Project URL: https://github.com/your-org/your-repo

    Build Triggers
    ☑ GitHub hook trigger for GITScm polling

    Pipeline
    Definition : Pipeline script from SCM
    SCM        : Git
    Repository URL: https://github.com/your-org/your-repo.git
    Credentials: (GitHub Token 선택)
    Branch     : */main
    Script Path: Jenkinsfile
    ```
4. **Save**

### GitHub Webhook 설정
- GitHub 저장소 > Settings > Webhooks > **Add webhook**에서 아래와 같이 입력
```
Payload URL  : http://<Jenkins서버IP>:8080/github-webhook/
Content type : application/json
Secret       : (비워두거나 Jenkins에서 설정한 Secret)
Trigger      : ☑ Just the push event
Active       : ☑
```
> Jenkins 서버가 외부에서 접근 불가한 경우, **ngrok** 또는 **Nginx 리버스 프록시**를 사용해 포트를 노출해야 함
