### 웹 에디터 내 이미지 삽입 구조
- 게시글 본문(content)에 이미지가 "중간에 렌더링되는" 형태는 대부분 **에디터 기반 이미지 업로드** 구조를 사용
- 이 경우 이미지 파일은 모델과 직접적인 N:1 관계로 관리할 필요가 없음
    1. 일단 이미지만 업로드
    2. URL을 에디터에 반환
    3. 에디터가 content HTML에 삽입
    4. 최종 저장 시 content 안에 이미지 URL이 포함됨
- 따라서 이미지 업로드 요청은 별도 API로 수행

### 업로드된 이미지의 모델 구성
```python
class UploadedImage(models.Model):
    image = models.ImageField(upload_to="editor/")
    uploaded_at = models.DateTimeField(auto_now_add=True)
```
- 게시글 model을 참조할 필요 없음
- content에 URL이 있기 때문에 게시글과의 관계는 자연스럽게 성립됨
