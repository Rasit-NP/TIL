# Difference Array

## 개요
- **구간 업데이트(range update)** 연산을 $O(1)$에 처리하기 위한 전처리 기법
- 특정 구간 $[l, r]$의 모든 원소에 동일한 값을 더하거나 빼는 연산이 반복될 때, 매번 $O(n)$을 소비하는 대신 전체 쿼리를 $O(q+n)$에 처리할 수 있음
- 적용 조건
    - 구간에 동일한 값을 일괄 적용하는 업데이트가 다수 존재
    - 업데이트 이후 배열 전체(또는 특정 인덱스)의 값을 조회
- 세그먼트 트리나 BIT가 점 업데이트/구간 쿼리에 강한 자료 구조라면, 차분 배열은 **구간 업데이트/점 쿼리 또는 전체 복원**에 특화된 가볍고 빠른 도구

## 기본 아이디어
### 1차원 차분 배열
- 원본 배열 `A[0,,n-1]`에 대해 차분 배열 `D`를 다음과 같이 정의
    ```
    D[0] = A[0]
    D[i] = A[i] - A[i-1]
    ```
- 이 정의에 의해 `A`는 `D`의 **prefix sum(누적 합)**
    ```
    A[i] = D[0] + D[1] + ... + D[i]
    ```
- 구간 업데이트의 핵심 원리
    - 구간 $[l, r]$의 모든 원소에 `+v`를 적용하고 싶다면, 차분 배열에서는 딱 두 지점만 수정
        ```
        D[l] += v;
        D[r+1] -= v;
        ```
    - 모든 업데이트를 마친 뒤, `D`의 prefix sum을 취하면 최종 배열 `A`를 $O(n)$에 복원할 수 있음
|   단계    |   시간 복잡도 |
|----------|--------------|
|   차분 배열 초기화    |   $O(n)$  |
|   구간 업데이트 1회   |   $O(1)$  |
|   q회 업데이트 후 배열 복원   |   $O(n)$  |
|   전체    |   $O(n+q)$    |

### 2차원 차분 배열
- 2차원 차분 배열 `A[n][m]`에서 부분 직사각형의 영역 `(r1, c1) ~ (r2, c2)` 전체에 `+v`를 적용하는 문제로 확장됨
- 2차원 차분 배열 `D[i][j]`를 다음과 같이 정의
    ```
    D[i][j] = A[i][j] - A[i-1][j] - A[i][j-1] + A[i-1][j-1]
    ```
- **직사각형 구간 업데이트**는 네 꼭지점만 수정
    ```
    D[r1][c1]     += v
    D[r1][c2+1]   -= v
    D[r2+1][c1]   -= v
    D[r2+1][c2+1] += v
    ```
- 복원은 `D`에 대해 **행 방향, 열방향**으로 순서대로 prefix sum을 두 번 취하면 됨

## 예시 코드
### 1차원 차분 배열
```cpp
# include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(NULL);  cout.tie(NULL);

    int n, q;   cin >> n >> q;

    vector<long long> A(n+1), D(n+2, 0);

    // 원본 배열 입력 및 차분 배열 초기화
    for (int i=1; i<=n; i++){
        cin >> A[i];
        D[i] = A[i] - A[i-1];
    }

    // 구간 업데이트 쿼리
    while (q--){
        int l, r;
        long long v;
        cin >> l >> r >> v;

        D[l] += v;
        D[r+1] -= v;
    }

    // 차분 배열로부터 최종 배열 복원
    for (int i=1; i<=n; i++){
        D[i] += D[i-1];
    }

    // 결과 출력
    for (int i=1; i<=n; i++){
        cout << D[i] << " \n"[i==n];
    }

    return 0;
}
```

### 2차원 차분 배열
```cpp
# include <bits/stdc++.h>
using namespace std;

int main(void){
    ios::sync_with_stdio(false);
    cin.tie(NULL);  cout.tie(NULL);

    int n, m, q;    cin >> n >> m >> q;

    // D를 (n+2) x (m+2)로 선언해 경계 초과를 안전하게 처리
    vector<vector<long long>> D(n+2, vector<long long>(m+2, 0));

    // 구간 업데이트
    while (q--){
        int r1, c1, r2, c2;
        long long v;
        cin >> r1 >> c1 >> r2 >> c2 >> v;

        D[r1][c1]           += v;
        D[r1][c2 + 1]       -= v;
        D[r2 + 1][c1]       -= v;
        D[r2 + 1][c2 + 1]   += v;
    }

    // 복원 1단계
    for (int i=1; i<=n; i++){
        for (int j=1; j<=m; j++){
            D[i][j] += D[i][j-1];
        }
    }

    // 복원 2단계
    for (int j=1; j<=m; j++){
        for (int i=1; i<=n; i++){
            D[i][j] += D[i-1][j];
        }
    }

    // 결과 출력
    for (int i=1; i<=n; i++){
        for (int j=1; j<=m; j++){
            cout << D[i][j];
            if (j<m)    cout << ' ';
        }
        cout << '\n';
    }

    return 0;
}
```

## 활용 팁 및 주의 사항
### 활용 팁
1. 배열 크기는 항상 여유 있게 잡기
    - `D[r+1]` 또는 `D[r2+1][c2+1]` 접근이 발생하므로, 항상 배열 크기를 `n+2`로 선언하는 습관 들이기
    - 경계값 처리 실수가 가장 흔한 버그 원인
2. 업데이트 중간에 점 쿼리가 필요한 경우
    - 차분 배열은 기본적으로 "모든 업데이트 후 일괄 복원"하는 구조
    - 업데이트 도중 특정 인덱스의 현재 값을 자주 조회해야 한다면, BIT(펜윅 트리) 기반의 구간 업데이트 + 점 쿼리 기법을 고려
3. 오프라인 처리와의 결합
    - 쿼리를 모두 읽은 뒤 한꺼번에 처리하는 **오프라인 알고리즘**과 잘 어울림
    - 좌표 압축(coordinate compression)과 함께 사용하면 희소(sparse)한 대형 배열에도 적용 가능
4. 차분 배열 + 이분 탐색
    - 최종 배열에서 "값이 0 이상인 구간의 최대 길이"같은 문제는 차분 배열로 복원 후 투 포인터나 이분 탐색을 결합할 수 있음

### 주의 사항
1. 자료형 오버플로우
    - 업데이트 값 `v`와 쿼리 수 `q`에 따라 누적 값이 int 범위를 초과할 수 있음
    - `long long`을 기본으로 사용할 것
2. 0-indexed vs. 1-indexed 혼용
    - `D[r+1]`이나 `D[r2+1][c2+1]` 접근 시, 인덱스 체계가 뒤섞이면 한 칸씩 밀리는 버그가 발생
    - 코드 전체에서 인덱스 기준을 통일
3. 구간 업데이트만 가능, 구간 쿼리는 불가
    - **구간 합 쿼리**까지 처리하려면 차분 배열을 두 번 중첩한 **2차 차분 배열** 기법(복원 후 $O(1)$)이나 세그먼트 트리 lazy propagation($O(\log n)$)이 필요
4. 복원은 반드시 한 번만
    - prefix sum 복원 후에 다시 업데이트를 추가하면 결과가 틀림
    - 차분 배열은 "모든 업데이트 → 한 번 복원 → 결과 사용"의 단방향 흐름으로 사용해야 함

|----------|--------------|----------|-----------|
