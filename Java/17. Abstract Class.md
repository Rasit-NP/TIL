# 추상 클래스
```java
abstract class ClassName {
    abstract void methodName();
}
```
- 클래스를 설계도에 비유한다면, 추상 클래스는 미완성 설계도
- 클래스가 미완성이라는 것은 미완성 메서드를 포함하고 있다는 것을 의미함
- 추상 클래스로 인스턴스는 생성할 수 없음
    - 상속을 통해서 자손 클래스에 의해서만 완성될 수 있음
- 추상 클래스는 추상 메서드를 포함하고 있다는 것을 제외하고는 일반 클래스와 다르지 않음
    - 생성자와 멤버 변수, 메서드를 가짐

## 추상 메서드(abstract method)
- 선언부만 작성하고 구현부는 작성하지 않은 채로 남겨둔 것
- 메서드의 내용이 상속받는 클래스에 따라 달라질 수 있어 조상에는 선언부만 작성
    - 주석을 덧붙여 어떤 기능을 수행할 목적으로 작성되었는지를 기술
    - 실제 내용은 상속받는 자손에서 구현
- 추상 메서드 역시 키워드 `abstract`를 붙임
- 구현부가 없으므로 괄호(`{}`) 대신 `;`를 붙임
- 선언부에서는 메서드의 이름과 필요한 매개변수, 반환 타입을 결정

## 추상 클래스 작성
- 여러 클래스에 공통적으로 사용될 수 있는 클래스를 바로 작성
- 혹은, 기존 클래스의 공통적인 부분을 뽑아서 추상 클래스로 만들어 상속하도록 함
- 추상화
    - 기존의 클래스의 공통 부분을 뽑아내서 조상 클래스를 만드는 것
