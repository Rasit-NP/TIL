# 클래스와 객체
## 클래스와 객체의 정의와 용도
- 클래스란 **객체를 정의한 것** 또는 **객체의 설계도**라고 할 수 있음
- 클래스는 객체를 생성하는데 사용되며, 객체는 클래스에 정의된 대로 생성됨
- 클래스를 잘 만들어 놓으면, 매번 객체를 생성할 때마다 어떻게 객체를 만들어야 할 지를 고민하지 않아도 됨
- JDK에서는 개발을 위해 많은 수의 유용한 클래스(Java API)를 기본적으로 제공하고 있음
    - 우리는 이 클래스들을 이용해서 원하는 기능의 프로그램을 보다 쉽게 작성할 수 있음

## 객체와 인스턴스
- 클래스로부터 객체를 만드는 과정을 클래스의 **인스턴스화(instantiate)**라고 하며, 어떤 클래스로부터 만들어진 객체를 그 클래스의 **인스턴스(instance)**라고 함
- 인스턴스와 객체는 같은 의미이므로 두 용어의 사용을 엄격히 구분할 필요는 없음
    - 다만, 문맥에 따라 구별하여 사용하는 것이 좋음

## 객체의 구성 요소 - 속성과 기능
```java
class TV {
    String color;
    boolean power;
    int channel;

    void power()        {power = !power;}
    void channelUp()    {channel++;}
    void channelDown()  {channel--;}
}
```
- 객체는 속성과 기능, 두 종류의 구성요소로 이루어져 있으며, 일반적으로 객체는 다수의 속성과 기능을 가짐
- 즉, 객체는 속성과 기능의 집합
- 속성 대신 **멤버 변수**, 기능 대신 **메서드**라는 이름을 사용

## 인스턴스의 생성과 사용
```java
Tv t;
t = new TV();

t.channel = 7;
t.channelDown();
```
- 인스턴스는 참조 변수를 통해서만 다룰 수 있으며, 참조 변수의 타입은 인스턴스의 타입과 일치해야 함

## 객체 배열
```java
Tv[] tvArr = new Tv[3];
```
- 다수의 객체를 다뤄야 할 때, 배열을 이용하면 편리함
- 객체 역시 배열로 다루는 것이 가능하며, 이를 **객체 배열**이라고 함
    - 배열에 객체가 저장되는 것이 아닌, 객체의 주소가 저장되는 것
- 모든 배열이 그렇듯이 객체 배열도 같은 타입의 객체만 저장할 수 있음
    - **다형성**을 배운 후에는, 하나의 배열로 여러 종류의 객체를 다룰 수 있음
