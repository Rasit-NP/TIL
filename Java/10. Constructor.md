# 생성자
```java
class Card{
    Card() {
        ...
    }

    Card(Stringk, int num){
        ...
    }
}
```

## 생성자
- 인스턴스가 생성될 때 호출되는 **인스턴스 초기화 메서드**
    - 인스턴스 초기화 작업에 주로 사용되며, 인스턴스 생성 시에 실행되어야 할 작업을 위해서도 사용됨
    - 생성자도 일종의 메서드지만 반환값이 없다는 점이 일반적인 메서드와 다름
        - 생성자 앞에 반환값이 없음을 뜻하는 `void`를 붙이지 않음
- 생성자의 조건은 아래와 같음
    1. 생성자의 이름은 클래스의 이름과 같아야 함
    2. 생성자는 반환값이 없음
- 생성자도 오버로딩이 가능함
- 연산자 `new`가 인스턴스를 생성하는 것이지 생성자가 인스턴스를 생성하는 것이 아님
    - 생성자가 갖는 몇 가지 특징만 제외하면 일반적인 메서드와 다르지 않음

## 기본 생성자(default constructor)
- 모든 클래스에는 반드시 하나 이상의 생성자가 정의되어 있어야 함
    - 클래스에 생성자를 정의하지 않고도 인스턴스를 생성할 수 있었던 이유는 컴파일러가 **기본 생성자**를 제공하기 때문
    - 컴파일할 때, 소스 파일의 클래스에 생성자가 정의되지 않은 경우, 컴파일러는 자동적으로 아래와 같은 내용의 기본 생성자를 추가함
        ```java
        ClassName() {}
        ```
    - 인스턴스 초기화 작업이 요구되지 않는다면 생성자를 정의하지 않고 기본 생성자를 사용해도 됨

## 매개 변수가 있는 생성자
```java
class Car {
    String color;
    String gearType;
    int door;

    Car() {}
    Car(String c, String g, int d){
        color = c;
        gearType = g;
        door = d;
    }
}

Car c = new Car("white", "auto", 4);
```
- 생성자도 메서드처럼 매개 변수를 선언하여 호출 시 값을 넘겨받아서 인스턴스의 초기화 작업에 사용할 수 있음
- 인스턴스마다 각기 다른 값으로 초기화되어야 하는 경우가 많기 때문에 매개 변수를 사용한 초기화는 매우 유용
- 매개 변수가 있는 생성자를 이용하면 인스턴스를 생성하는 동시에 원하는 값으로 초기화를 할 수 있음
    - 인스턴스를 생성한 다음에 인스턴스 변수의 값을 변경하는 것보다 코드가 짧고 수정에 용이함

## `this()`, `this`
- 아래 조건을 만족시키면, 생성자도 서로 호출이 가능함
    - 생성자의 이름으로 클래스 이름 대신 `this` 사용
    - 다른 생성자를 호출할 때 첫 줄에서 호출
- `this`는 참조 변수이며 인스턴스 자신을 가리킴
    - `this`를 사용할 수 있는 것은 인스턴스 멤버 뿐
    - `static` 메서드에서는 `this` 역시 사용할 수 없음
    - 모든 인스턴스 메서드에 지역 변수로 숨겨진 채로 존재함

## 생성자를 이용한 인스턴스의 복사
```java
Car(Car c){
    this(c.color, c.gearType, c.door);
}
```
