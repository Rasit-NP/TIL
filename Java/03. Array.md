# 배열
## 배열
- 같은 타입의 여러 변수를 하나로 묶은 것
- 변수의 선언과 달리 다뤄야 할 데이터의 수가 아무리 많아도 배열의 길이만 바꾸면 됨

## 배열의 선언과 생성
```java
int[] score = new int[4];
int score[] = new int[4];
```

### 배열의 생성
- 배열을 선언하는 것은 단지 생성된 배열을 다루기 위한 참조 변수를 위한 공간이 만들어질 뿐
- 배열을 생성해야만 값을 저장할 공간이 만들어짐
- 배열을 생성하려면 연산자 `new`와 함께 배열의 타입과 길이를 지정해 주어야 함

## 배열의 길이와 인덱스
- 생성된 배열의 각 저장 공간을 **배열의 요소**라고 하며, `array[index]`의 형태로 배열의 요소에 접근함

### 배열의 길이
- 배열의 길이는 배열 요소의 개수, 즉 값을 저장할 수 있는 공간의 개수
- 배열의 길이는 양의 정수여야 하며, 최대값은 `int` 타입의 최대값인 약 20억
- 길이가 0인 배열도 생성 가능

### `arr.length`
```java
int[] arr = new int[5];
int tmp = arr.length;
```
- 배열의 길이는 `arr.length`를 통해 알 수 있음
- 배열은 한 번 생성하면 길이를 변경할 수 없기 때문에, `arr.length`는 상수임
    - 값을 읽을 수만 있을 뿐, 변경할 수 없음

### 배열의 길이 변경하기
- 배열은 한 번 선언되고 나면 길이를 변경할 수 없음
- 배열에 저장할 공간이 부족할 경우
    - 더 큰 길이의 새로운 배열을 생성
    - 기존의 배열에 저장된 값들을 새로운 배열에 복사
- 이런 작업들은 비용이 많이 들기 때문에, 처음부터 배열의 길이를 넉넉하게 잡아줘서 새로 배열을 생성해야 하는 일이 가능한 적게 발생하도록 해야 함

## 배열의 초기화
```java
int[] score = new int[] {50, 60, 70, 80, 90};
int[] score = {50, 60, 70, 80, 90};
```
- 배열은 생성과 동시에 자동으로 자신의 타입에 해당하는 기본값으로 초기화됨
- 원하는 값을 저장하려면 각 요소마다 값을 지정해 줘야 함
- `Java`에서는 배열을 간단히 초기화할 수 있는 방법을 제공
    - 다만, 배열의 선언과 생성을 따로 하는 경우에는 `new int[]`를 생략할 수 없음

### 배열의 출력
```java
int[] iArr = {100, 95, 80, 70, 60};

for (int i=0; i<iArr.length; i++){
    System.out.println(iArr[i]);
}

System.out.println(Arrays.toString(iArr));
```
- `for`문을 활용하여 배열에 저장된 값을 확인 가능
- `Arrays.toString(arr)` 메서드를 사용해서도 확인 가능
- `iArr`의 값을 바로 출력하게 되면, `타입@주소`의 형식이 출력됨
    - 예외적으로 `char` 배열은 `println` 메서드로 출력하면 각 요소가 구분자 없이 그대로 출력됨

## 배열의 복사
- 배열을 복사하는 방법에는 두 가지가 있음
- `for` 문을 이용한 방법
    ```java
    int[] arr = {1, 2, 3, 4, 5};
    int[] tmp = new int[arr.length*2];
    for (int i=0; i < arr.length; i++){
        tmp[i] = arr[i];
    }
    arr = tmp;
    ```

### `System.arraycopy()`
```java
System.arraycopy(num, 0, newNum, 0, num.length);
```
- `arraycopy()`는 지정된 범위의 값들을 한 번에 통째로 복사
    - 각 요소들이 연속적으로 저장되어 있다는 배열의 특성 때문에 이렇게 처리하는 것이 가능
- 배열의 복사는 `for` 문보다 `System.arraycopy()`를 사용하는 것이 빠름
